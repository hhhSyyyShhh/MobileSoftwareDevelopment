# Lab7：构建可滚动课程网格应用 实验报告

## 一、应用整体结构说明
本次实验基于 Jetpack Compose 框架开发课程网格展示应用，采用数据与 UI 分离的模块化设计。应用核心分为三个部分：Topic.kt 数据类负责定义课程数据模型，DataSource.kt 单例对象负责提供 24 个课程主题的静态数据，MainActivity.kt 实现可滚动网格布局与课程卡片 UI 组件。资源文件严格按照要求存放，图片与图标置于 res/drawable，字符串资源置于 res/values/strings.xml，整体结构清晰规范。

## 二、Topic 数据类的字段设计与选择理由
Topic 数据类用于封装课程主题的核心数据，包含三个字段：nameRes 为字符串资源 ID，使用 @StringRes 注解，用于引用课程名称，避免硬编码；courseCount 为整型，存储课程数量；imageRes 为图片资源 ID，使用 @DrawableRes 注解，用于引用课程配图。字段设计完全匹配卡片展示需求，使用资源注解符合 Android 开发规范，保证资源引用安全，结构简洁且适配 Compose 数据渲染。

## 三、卡片布局实现思路
课程卡片以 Card 为外层容器，内部使用 Row 实现横向布局。左侧为 68dp×68dp 的正方形图片，保证 1:1 比例显示；右侧为 Column 垂直布局的文字区域，设置 16dp 内边距。文字区域上方为课程名称，使用 bodyMedium 字体样式，下方为图标与课程数量，使用 labelMedium 字体样式。标题与图标区域间距 8dp，图标与数字间距 8dp，所有尺寸、样式、间距均严格遵循实验 UI 规格，组件封装后可循环复用。

## 四、网格布局实现思路
使用 LazyVerticalGrid 实现两列可垂直滚动的网格布局，通过 GridCells.Fixed(2) 固定列数为 2。为网格设置 fillMaxSize 修饰符使其占满屏幕，保证滚动功能生效，整体添加 8dp 内边距，卡片横向与纵向间距均设置为 8dp。通过 items 方法遍历数据源中的 24 条数据，自动生成所有课程卡片。LazyVerticalGrid 为惰性加载组件，仅渲染可见区域内容，性能高效，内容超出屏幕后自动支持上下滑动。

## 五、遇到的问题与解决过程
1. 界面无法上下滑动：原因是 LazyVerticalGrid 未设置全屏修饰符，添加 fillMaxSize() 后滚动功能正常。
2. 代码爆红、资源链接失败：原因是字符串资源不完整、图片文件名不匹配，统一图片名称并使用实验指定的字符串资源后修复。
3. 卡片布局不符合要求：修正图片尺寸、内边距、字体样式和间距，严格按照 UI 规格实现。
4. 模拟器无法拖动滑动：属于运行环境限制，在 Android 模拟器中运行即可正常滑动。

## 六、实验总结
本次实验完成了可滚动课程网格应用的开发，成功实现 24 个课程主题的两列网格展示，界面支持垂直滚动，布局、样式、功能均符合实验要求。通过实验掌握了数据类设计、单例数据源管理、LazyVerticalGrid 网格布局使用、Compose 基础组件嵌套等核心技能，提升了按照 UI 规范开发和独立解决问题的能力，顺利完成了实验全部目标。
