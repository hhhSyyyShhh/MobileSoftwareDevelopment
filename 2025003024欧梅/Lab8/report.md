# Lab8 Superheroes 实验报告

## 一、实验目标
本次实验使用 Jetpack Compose 构建一个超级英雄列表应用，实现以下核心目标：
1. 完成 6 个超级英雄的信息展示，包含名称、描述与对应图片。
2. 实现可滚动列表（LazyColumn），优化列表布局与间距。
3. 自定义 Material3 主题，包含颜色、字体（Cabin）、形状。
4. 实现**浅色主题 + 深色主题**双模式自动适配，界面完全匹配官方效果图。
5. 完成顶部应用栏、沉浸式状态栏、卡片圆角与阴影效果。

## 二、项目结构
本项目采用模块化、分层设计，结构清晰规范：
- **Hero.kt**：数据类，封装英雄名称、描述、图片资源。
- **HeroesRepository.kt**：数据源，提供 6 个超级英雄静态数据。
- **HeroesScreen.kt**：UI 界面，包含列表 HeroesList 与列表项 HeroItem。
- **MainActivity.kt**：应用入口，搭建 Scaffold、TopAppBar。
- **ui.theme 包**：Color.kt、Theme.kt、Type.kt、Shape.kt 统一管理主题。
- **资源文件**：drawable 图片、font 自定义字体、strings 字符串。

## 三、核心功能实现

### 1. 数据模型设计
Hero 数据类包含三个字段：
- nameRes：英雄名称字符串资源
- descriptionRes：英雄描述字符串资源
- imageRes：英雄图片资源
使用资源注解 @StringRes、@DrawableRes 保证类型安全，符合 Android 开发规范。

### 2. 列表与布局实现
- 使用 LazyColumn 实现惰性加载列表，支持上下滑动。
- 列表整体内边距 16dp，列表项间距 8dp。
- 每个列表项使用 Card 容器，圆角 16dp，阴影 2dp。
- 内部采用 Row 布局：**文字在左，图片在右**，垂直居中对齐。
- 图片尺寸 72dp，圆角 8dp，与效果图完全一致。

### 3. 浅色主题实现
- 背景色：#FFFFFBFE（纯净浅白）
- 卡片色：#FFE3E1DC（浅灰卡片）
- 文字色：#FF1C1B1F（深黑文字）
- 整体明亮清晰，符合官方浅色效果图样式。

### 4. 深色主题实现
- 背景色：#FF1C1B1F（深黑灰）
- 卡片色：#FF484645（中度深灰卡片）
- 文字色：#FFE6E1E5（浅白文字）
- 自动适配系统深色模式，也可代码强制开启。
- 深浅色切换时，文字、卡片、背景自动对应配色，无样式错乱。

### 5. 自定义主题配置
- 字体：导入 Cabin-Regular、Cabin-Bold，全局统一使用。
- 形状：图片圆角 8dp，卡片圆角 16dp。
- 顶部栏标题居中，字体放大加粗，与效果图一致。
- 状态栏沉浸式适配，界面全屏无遮挡。

## 五、遇到的问题与解决方案
1. **函数重复定义报错**
   - 原因：MainActivity 与 HeroesScreen 同时定义了 HeroesList、HeroItem。
   - 解决：仅在 HeroesScreen.kt 保留 UI 组件，MainActivity 只负责调用。

2. **布局错位、图片位置不对**
   - 原因：图片与文字顺序错误，未设置权重与对齐。
   - 解决：调整顺序为文字左、图片右，添加 weight 与居中对齐。

3. **深色模式颜色不匹配**
   - 原因：配色与官方效果图不一致。
   - 解决：精确配置浅/深色背景、卡片、文字颜色，实现 1:1 还原。

4. **导入缺失导致爆红**
   - 原因：Modifier、padding、layout 等依赖未导入。
   - 解决：补充完整导入语句，代码可正常编译运行。
