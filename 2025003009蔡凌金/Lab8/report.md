# Lab8 超级英雄应用 实验报告

## 一、应用整体结构说明
本次实验基于Jetpack Compose与Material 3构建超级英雄列表应用，整体采用**分层结构**设计：
1. **数据层**：包含`Hero`数据类与`HeroesRepository`静态数据源，实现数据封装与解耦；
2. **UI界面层**：拆分独立组件，包含英雄列表`HeroesList`、单项组件`HeroItem`，实现界面复用；
3. **主题层**：自定义颜色、字体、形状三套样式，统一全局UI风格；
4. **框架层**：使用`Scaffold`搭建整体结构，集成顶部应用栏，处理页面边距与系统状态栏适配。

整体结构模块化，组件独立化，符合Compose声明式UI开发规范。

## 二、Hero数据类字段设计与理由
定义`Hero`数据类，包含三个资源类型字段：
1. `@StringRes val nameRes`：存储英雄名称资源ID；
2. `@StringRes val descriptionRes`：存储英雄介绍文本资源ID；
3. `@DrawableRes val imageRes`：存储英雄图片资源ID。

**设计理由**：
- 使用资源注解`@StringRes`、`@DrawableRes`，避免传入错误数值，提升代码安全性；
- 统一使用资源ID而非文本、图片实体，便于本地化、资源统一管理；
- 数据类结构简洁，适配列表渲染需求，满足界面展示需求。

## 三、HeroesRepository数据源组织方式
`HeroesRepository`采用**单例object**模式管理静态数据：
1. 以全局集合`List`存储所有英雄数据；
2. 一次性初始化6个英雄对象，关联对应字符串与图片资源；
3. 数据源集中管理，与UI完全分离，便于后期修改、增删数据。

无需动态数据，采用静态列表实现，满足实验展示需求。

## 四、英雄列表项布局实现思路
单个英雄卡片`HeroItem`采用**Row+Column**组合布局：
1. 外层使用`Card`组件，搭配自定义圆角样式，实现卡片效果；
2. 横向`Row`布局：左侧为文本区域，右侧为图片；
3. 左侧`Column`垂直排列：英雄名称、英雄简介；
4. 右侧设置固定尺寸图片，添加圆角裁剪与`ContentScale`控制图片填充；
5. 统一设置内边距、组件间距，严格遵循界面尺寸规范。

布局层次清晰，结构简单，符合Material 3设计规范。

## 五、LazyColumn列表实现和间距说明
1. 采用`LazyColumn`实现**懒加载滚动列表**，高效渲染列表，减少资源占用；
2. 启用`contentPadding`设置列表整体四周留白；
3. 使用`Arrangement.spacedBy`控制列表项之间的垂直间距；
4. 调用`items()`遍历数据源，循环渲染单个`HeroItem`组件；
5. 结合Scaffold的`innerPadding`，避免顶部栏遮挡列表内容。

## 六、Material主题配置说明
### 1. 颜色
自定义浅色、深色两套`ColorScheme`，适配系统深浅模式，保证文字与背景对比度。
### 2. 字体
导入自定义Cabin字体，配置`FontFamily`，定义多级文本样式（标题、正文），全局统一字体。
### 3. 形状
自定义`Shapes`，设置卡片圆角、图片圆角，统一界面圆角风格。
### 4. 全局整合
通过`MaterialTheme`整合颜色、字体、形状，全局组件自动引用主题样式。

## 七、顶部应用栏与状态栏处理
1. **顶部应用栏**：使用`TopAppBar`，标题居中，采用自定义标题字体样式；
2. 依托`Scaffold`组件统一管理顶部栏与内容区域；
3. **状态栏适配**：设置状态栏透明，结合深浅模式动态切换状态栏图标颜色；
4. 利用`innerPadding`补偿边距，防止内容被顶部栏、系统栏遮挡，界面视觉完整。

## 八、遇到的问题与解决过程
1. **问题：代码提示资源引用错误、缺少导入**
   解决：手动添加Compose布局、资源相关导入，使用IDE快速补全解决。

2. **问题：单个英雄图片无法显示**
   解决：检查drawable文件夹资源文件完整性，修正图片尺寸与布局约束，调整图片裁剪参数。

3. **问题：列表内容被顶部栏遮挡**
   解决：将Scaffold返回的`innerPadding`传入列表组件，修正边距。

4. **问题：数据类出现无效警告**
   解决：修改资源注解写法，使用`@get:`注解，消除语法警告。

5. **问题：深浅主题切换显示异常**
   解决：优化主题配置，动态调整状态栏颜色，保证深浅模式下文字清晰。
