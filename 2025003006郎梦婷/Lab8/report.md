# Lab8 构建超级英雄列表应用 实验报告

## 一、应用整体结构说明
本项目基于 Android Jetpack Compose 与 Material 3 开发，采用分层架构设计，整体结构清晰，分为数据层、UI 层、主题层三部分。
数据层负责定义数据结构与提供数据源，UI 层实现列表与列表项展示，主题层负责颜色、字体、形状的统一管理，并支持浅色与深色主题自动切换。项目结构规范，符合 Android 开发最佳实践，满足实验要求。

## 二、Hero 数据类字段设计与理由
Hero 数据类用于封装单个超级英雄的信息，包含三个字段：英雄名称资源 ID、英雄描述资源 ID、英雄图片资源 ID。全部使用资源 ID 而非硬编码，便于多语言适配、主题切换与资源统一管理。
该设计简洁完整，能够准确描述列表所需数据，符合实验要求与 Android 开发规范。

## 三、HeroesRepository 数据源组织方式
使用单例对象 HeroesRepository 集中管理所有英雄静态数据，全局唯一，方便在任意界面获取数据，实现数据与 UI 解耦。
数据源包含六个超级英雄信息，统一维护、易于扩展，满足实验对集中式数据源的要求。

## 四、英雄列表项布局实现思路
列表项采用 Card + Row + Column + Image 组合实现。
外层使用卡片提供圆角与背景色，内部使用 Row 实现文字与图片水平排列，Column 展示英雄名称与描述，右侧展示固定大小的圆角图片。
布局严格按照实验规格实现，包括卡片高度、内边距、图片大小、圆角、文字样式，与官方效果图完全一致。

## 五、LazyColumn 列表实现和间距配置说明
使用 LazyColumn 实现高性能滚动列表，设置整体内容内边距 16dp，列表项之间垂直间距 8dp，符合实验给出的布局规范。
通过 items 方法遍历数据源，自动渲染所有列表项，实现数据驱动 UI 展示。

## 六、Material 主题配置说明
本项目完成了完整的 Material 3 主题定制：
1. 颜色：配置浅色与深色两套配色方案，卡片、背景、文字颜色可自动切换。
2. 字体：引入 Cabin 字体，设置标题与正文样式，统一界面文字风格。
3. 形状：自定义圆角大小，卡片使用中等圆角，图片使用小圆角。
主题可跟随系统自动切换，满足浅色与深色模式展示要求。

## 七、顶部应用栏和状态栏处理说明
使用 Scaffold 与居中顶部应用栏实现页面结构，标题居中显示，使用指定文字样式。
状态栏设置为透明沉浸式，根据浅色/深色主题自动调整图标颜色，使界面整体美观统一，无色块断层，符合实验要求。

## 八、遇到的问题与解决过程
1. 问题：浅色与深色主题无法自动切换。
解决：配置两套颜色方案，根据系统主题自动选择。

2. 问题：列表项布局不整齐、文字与图片不对齐。
解决：固定卡片高度，使用 Row 垂直居中对齐。

3. 问题：状态栏颜色不协调、界面突兀。
解决：设置透明状态栏并自动适配图标深浅。

4. 问题：卡片颜色不符合效果图。
解决：使用主题颜色 surfaceVariant，实现自动适配。

## 运行结果
应用成功展示六个超级英雄信息，列表滚动流畅，布局规范统一。
浅色主题与深色主题切换正常，界面风格与实验参考图一致，顶部栏、字体、颜色、间距均满足要求，项目功能完整，达到实验全部目标。
