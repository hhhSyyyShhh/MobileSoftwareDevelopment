## 一、实验目的
1. 理解 Jetpack Compose 中 Navigation 导航组件的作用与核心概念。
2. 掌握使用 NavHost、NavController 实现多页面跳转与返回栈管理的方法。
3. 学会使用枚举类统一管理应用页面路由与标题资源。
4. 实现顶部应用栏（TopAppBar）的动态适配，支持页面标题切换与返回按钮功能。
5. 结合 ViewModel 实现跨页面数据共享，理解导航与状态管理的结合方式。
6. 熟悉 Compose 项目的导航流程、依赖配置与项目构建。

## 二、实验原理
Navigation 是 Android Jetpack 提供的导航组件，专门用于处理应用内的页面跳转与导航栈管理，能在 Compose 中便捷实现单 Activity 多页面的应用架构。它通过 NavHostController 管理页面栈，配合 NavHost 定义所有可导航页面，实现页面间的跳转、返回与参数传递。

在 Compose 项目中，使用 `rememberNavController()` 获取导航控制器实例，通过 `NavHost` 的 `composable` 为每个页面配置路由，配合 `navigate()` 与 `navigateUp()` 实现页面跳转与返回。同时使用枚举类统一管理页面路由与标题，让导航逻辑更清晰易维护。

本实验将项目拆分为三层：
1.  导航层：NavHost 与 NavController 统一管理页面跳转、返回栈与路由配置。
2.  界面层：各页面组合函数（StartOrderScreen、EntreeMenuScreen 等）负责页面布局、控件展示与事件回调。
3.  状态层：OrderViewModel 统一管理订单数据，跨页面共享状态，实现数据与导航逻辑的解耦。

## 三、实验步骤
1.  创建 Compose 项目，添加 Navigation、ViewModel 相关依赖配置。
2.  定义 `LunchTrayScreen` 枚举类，统一管理所有页面路由名称与标题资源。
3.  实现 `LunchTrayAppBar` 顶部导航栏组件，支持根据当前页面动态显示标题与返回按钮。
4.  在主函数中初始化 NavController，通过 `currentBackStackEntryAsState()` 获取当前页面状态。
5.  使用 NavHost 配置各页面路由，绑定对应的界面组合函数。
6.  为每个页面配置跳转逻辑：开始页 → 主菜页 → 配菜页 → 佐餐页 → 结账页。
7.  实现页面的取消与重置逻辑，点击取消按钮时清空订单并返回至开始页。
8.  运行项目，验证页面跳转、返回、标题切换与订单数据跨页面传递功能是否正常。

## 四、实验结果
1.  应用启动后正常进入开始页，点击“开始点餐”按钮可成功跳转到主菜选择页，顶部标题同步更新为“选择主菜”。
2.  在各菜单页面选择菜品后，点击“下一步”可跳转到下一级页面，数据可正常传递至 ViewModel。
3.  点击顶部返回按钮可返回上一级页面，导航栈状态正常，订单数据不会因页面返回而丢失。
4.  在任意页面点击“取消”按钮，订单数据会被重置，应用直接返回至开始页。
5.  进入结账页后，订单数据可正常展示，点击“提交订单”可重置订单并返回开始页。
6.  所有页面标题、导航栏状态与页面跳转逻辑均符合预期，无崩溃或页面跳转异常问题。
