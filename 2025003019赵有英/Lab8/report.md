# Lab8 实验报告：超级英雄列表应用

## 一、应用整体结构说明
本项目采用 Compose 单 Activity 架构，分为四层：
1.  **数据层**：通过 `Hero` 数据类和 `HeroesRepository` 管理静态英雄数据，与 UI 层解耦。
2.  **UI 组件层**：`HeroItem` 封装单个英雄卡片，`HeroesScreen` 用 `LazyColumn` 实现可滚动列表。
3.  **主题层**：通过自定义 Material 3 主题，配置了颜色、字体（Cabin）和形状，支持浅色/深色模式自动适配。
4.  **入口层**：`MainActivity` 用 `Scaffold` 整合顶部应用栏和列表，实现完整应用结构。

## 二、Hero 数据类设计说明
`Hero` 数据类包含三个字段：
- `nameRes` / `descriptionRes`：存储字符串资源 ID，方便多语言适配。
- `imageRes`：存储图片资源 ID，便于统一管理资源。

该设计让数据与具体资源解耦，方便后续修改和扩展。

## 三、HeroesRepository 数据源组织
使用 `object` 单例模式管理静态数据，将 6 个英雄信息统一存储在 `heroes` 列表中，UI 层只需调用 `HeroesRepository.heroes` 即可获取数据，实现了数据与界面的分离。

## 四、UI 组件实现说明
1.  **HeroItem 卡片组件**：
    - 使用 `Card` 实现卡片样式，添加圆角和阴影效果，提升层次感。
    - 通过 `Row` + `Column` 布局，将文本与图片横向排列，保证信息清晰可读。
    - 图片使用 `ContentScale.Crop` 裁剪，确保头像显示效果统一。

2.  **HeroesScreen 列表组件**：
    - 使用 `LazyColumn` 实现高性能列表，仅渲染屏幕可见项，提升滑动流畅度。
    - 通过 `Arrangement.spacedBy` 为列表项添加间距，优化界面布局。
    - 接收外部传入的 `PaddingValues`，适配 `Scaffold` 顶部栏的内边距。

## 五、主题与样式配置
1.  **颜色系统**：定义了亮色与深色两套配色方案，在不同主题下自动切换，保证文字与背景的对比度符合设计规范。
2.  **形状系统**：配置了统一的圆角样式，应用于卡片组件，保持界面风格一致。
3.  **字体系统**：引入自定义 Cabin 字体，配置标题与正文的字体大小和样式，构建专属排版体系。

## 六、运行效果展示
### 1. 浅色模式效果
应用在浅色主题下运行正常，背景为白色，卡片与文字对比度清晰，列表滑动流畅，所有英雄信息均正确展示。

### 2. 深色模式效果
应用在深色主题下自动适配，背景为深灰色，文字与卡片颜色同步切换，整体风格统一，无显示异常。

## 七、遇到的问题与解决
1.  **问题**：字体文件报红，无法引用。
    - **解决**：将字体文件名修改为全小写+下划线格式，符合 Android 资源命名规范。
2.  **问题**：`MainActivity` 出现函数重载冲突错误。
    - **解决**：清理冗余导入，使用标准的 `Scaffold` 组件写法，避免组件引用冲突。
3.  **问题**：模拟器启动后应用崩溃。
    - **解决**：关闭模拟器并重启，同时清理项目缓存后重新构建，解决临时状态异常问题。

## 八、实验总结
本次实验完成了一个基于 Jetpack Compose 的超级英雄列表应用，实现了数据模型、界面组件与主题系统的完整开发流程。通过本次实验，熟练掌握了 Compose 常用组件的组合使用、Android 资源文件的规范管理，以及数据与界面分离的开发思想，为后续 Android 应用开发打下了坚实基础。
