# Superheroes 应用开发报告

## 一、应用整体结构说明
本应用基于 Jetpack Compose 框架开发，采用 UI 与数据分离的设计思想。整体结构分为：应用入口 MainActivity、主题配置 SuperheroesTheme、主页面布局 SuperheroesApp、英雄列表 HeroesList、列表条目 HeroItem，以及数据模型 Hero 和数据源 HeroesRepository。各模块职责明确，结构分层清晰，便于维护和扩展。

## 二、Hero 数据类字段设计与理由
使用资源 id 而非硬编码字符串和图片，定义三个字段：名称资源、描述资源、图片资源。
采用 @DrawableRes 注解约束图片资源类型，保证编译安全；全部使用资源引用，方便多语言适配和后期统一修改，符合 Android 开发规范。

## 三、HeroesRepository 数据源组织方式
通过单例 object 对象统一管理英雄集合，内部使用 List 存储多个 Hero 实体。所有数据均引用字符串和图片资源文件，不写死文本和图片路径，实现数据与 UI 完全解耦，方便增删英雄条目。

## 四、英雄列表项布局实现思路
每个英雄条目使用 Card 卡片布局，内部采用 Row 水平排列：左侧为 Column 垂直展示英雄名称和描述，右侧固定尺寸展示英雄图片。使用 Spacer 设置间距，图片采用 Crop 缩放方式保证比例美观，整体符合 Material3 卡片设计风格。

## 五、LazyColumn 列表实现和间距配置说明
采用 LazyColumn 实现高性能滚动列表，仅加载屏幕可见条目，适合列表数据展示。设置整体内容内边距，每个列表项设置底部间距，使卡片之间分隔明显，页面布局整齐美观。

## 六、Material 主题配置说明（颜色、字体、形状）
项目使用 Material3 主题，内置浅色模式和暗黑模式两套配色；统一设置全局字体排版样式，标题、正文文本层级分明；主题内置统一圆角形状，Card、按钮等组件自动适配，全局视觉风格统一。

## 七、顶部应用栏和状态栏处理说明
使用 Scaffold 脚手架布局，自带 TopAppBar 顶部标题栏，自动预留内容内边距，避免内容被状态栏、标题栏遮挡。顶部栏右侧添加文字按钮，用于切换深浅主题，布局结构标准简洁。

## 八、浅色/暗黑模式切换实现逻辑
在 MainActivity 中使用 remember + mutableStateOf 记录主题状态，默认浅色模式。将状态和切换回调传递到顶层布局，点击顶部栏图标后状态取反，传入 SuperheroesTheme 实现全局一键切换浅色/暗黑模式，所有页面自动跟随主题刷新。
