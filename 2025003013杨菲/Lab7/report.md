# Lab7：构建可滚动课程网格应用

# Lab7 实验报告
## 1. 应用整体结构说明
本项目采用分层开发结构，划分为数据模型、数据源、UI 组合项三部分。
- model 包下的 Topic 数据类，用于封装单条课程的所有信息。
- data 包下的 DataSource 单例类，统一管理全部 24 条课程静态数据，实现数据与界面解耦。
- MainActivity 中编写卡片组件与网格布局组合函数，负责页面展示与滚动交互。
- 字符串、图片、图标统一放置在 res 资源目录中，通过资源引用进行加载使用。

## 2. Topic 数据类字段设计与理由
Topic 为数据类，包含三个核心字段：
- 名称资源ID：使用@StringRes注解，引用字符串资源，避免硬编码。
- 课程数量：整型数值，用于展示对应主题的课程总数。
- 图片资源ID：使用@DrawableRes注解，引用drawable图片资源。

选择资源ID作为参数，可统一管理项目资源，方便后期修改与维护，注解可以在编译阶段校验资源合法性，减少报错。

## 3. 卡片布局实现思路
单个课程卡片使用 Card 作为外层容器。
内部通过 Column 垂直排列，上方为课程图片，设置 1:1 固定宽高比保证正方形显示；下方为文字区域，整体设置 16dp 内边距。
文字区域内垂直放置课程名称文本，采用 bodyMedium 文字样式；再通过 Row 水平排列图标与课程数量文字，设置 8dp 间距，课程数量使用 labelMedium 样式，严格对照实验尺寸与字体规范完成布局。

## 4. 网格布局实现思路
页面核心使用 LazyVerticalGrid 实现可滚动网格。
通过 GridCells.Fixed(2) 固定为两列布局；
利用 spacedBy 设置卡片水平、垂直间距为 8dp；
contentPadding 设置网格整体外边距 8dp。
使用 items 遍历数据源集合，批量加载自定义课程卡片，LazyVerticalGrid 具备懒加载特性，支持长列表流畅垂直滚动。

## 5. 遇到的问题与解决过程
1. 项目提示 CoursesTheme 引用错误，原因是主题文件缺失，补全 ui/theme 下主题文件后报错解决。
2. drawable 目录编译报错，目录内存在无关无后缀文件，删除多余文件，仅保留图片与xml资源后正常构建。
3. 部分资源无法识别，检查图片文件名与代码引用名称保持小写统一，同步项目资源后正常显示。
4. 代码组件导入爆红，手动补全 Compose 相关导入语句，同步依赖后代码恢复正常。
